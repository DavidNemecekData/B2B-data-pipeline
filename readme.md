## 📖 Project Overview
This project demonstrates the end-to-end design and implementation of a data pipeline. It covers the lifecycle from ETL processing and relational modeling to data mart construction and business intelligence visualization, transforming highly unstructured corporate data into an automated Single Source of Truth.

**The Challenge:** The initial state consisted of B2B CRM and sales journal data stored in two massive MS Excel spreadsheets. This legacy setup suffered from high data redundancy, zero referential integrity, terminological inconsistency, a wide-format structure and key business statuses encoded solely via cell background colors rather than text.

*(You can see an illustration of the original data state in the [docs folder](./docs/)).*

### 🎯 Core Objectives:
1. **Design a strict relational model and build a PostgreSQL database** to eliminate data anomalies and redundancy.
2. **Clean and transform** matrix-style data into a vertical format (Tidy Data).
3. **Decode implicit metadata** extract cell colors into text statuses.
4. **Implement an analytical layer (dbt)** to standardize, test, and document SQL transformations.
5. **Develop an interactive dashboard (Power BI)** on top of the clean Star Schema.
6. **Ensure data anonymization** of production data for public portfolio presentation.

## 🏗️ Solution Architecture & Project Phases

### 1. Data Extraction & Cleaning (Excel/Power Query)
* **Visual Metadata Extraction:** Business statuses were hidden in cell colors. A macro function (`=GET.CELL(63, ...)`) was deployed to convert colors into numeric indices, which were later mapped to a `contact_status_enum`.
* **Structural Normalization (Unpivot):** Wide tables tracking chronological development (a new column for each month) were transformed into a Long format via Power Query. This ensures Schema Evolution without altering the database structure.
* **Cleaning & Mapping:** String trimming, deduplication, unification of terms, and standardizing data types.

### 2. Database Design & Load (PostgreSQL)
A relational model was designed to handle complex B2B relationships (including M:N tables like `company_specialization`). PostgreSQL features were utilized to ensure maximum data quality at the engine level:
* **Custom ENUM Types:** Strict value constraints (e.g., `field_enum`, `status_enum`).
* **Stored Generated Columns:** Automated calculation of final prices directly in the table definition based on list prices, discounts, and provisions.
* *(Explore the DDL and Database Architecture in the [sql_database](./sql_database/) folder).*

### 3. Anonymization (Mockaroo)
To comply with GDPR and protect corporate assets, production data was replaced with synthetic data using Mockaroo. Any logical inconsistencies generated by the randomization process were cleaned using SQL techniques prior to downstream modeling. *(Read more in the [data](./data/) folder).*

### 4. Transformation Layer (dbt)
A robust transformation layer was built on top of the relational model:
* **Staging:** Basic unification and casting.
* **Marts:** Creation of wide, denormalized views (Star Schema) optimized for rapid BI reporting.
* **Data Quality & Testing:** Implementation of automated generic and singular dbt tests to guarantee the integrity of data entering the final reports.
* *(Explore the models in the [dbt_project](./dbt_project/) folder).*

### 5. Reporting (Power BI)
The final layer consists of an interactive dashboard connected to the dbt marts, providing a comprehensive overview of financial performance, CRM conversion rates, and discount impacts across individual months and specializations.
